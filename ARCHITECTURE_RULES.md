# Правила архитектуры проекта: Станки-Иннос

Данный документ описывает жесткие стандарты файловой структуры для хранения страниц оборудования (Каталога), раздела Новостей и медиа-ассетов. Строгое соблюдение этой архитектуры (Flat-Tree / Silo) гарантирует чистоту проекта, удобство копирования компонентов и корректную SEO-индексацию.

## 1. Схема папок и маршрутизация (URL Routing)

Корневая папка `innos-site` должна содержать только общие (хаб) страницы. Любая вложенная сущность (Новость или Карточка Товара) должна лежать в собственной физической директории вместе со своими локальными ресурсами (`assets`).

```text
innos-site/
├── assets/                     # Глобальные ассеты сайта (общие логотипы, шрифты, стили)
├── news/                       # Раздел Новостей (Директория-обертка)
│   ├── news-1/                 #   - Папка конкретной новости (например: /news/exhibition-2024/)
│   │   ├── assets/             #     - Локальная папка с медиа только для этой новости
│   │   └── news-1.html         #     - Страница самой новости
│   └── news-2/                 #   - Папка следующей новости...
│       ├── assets/
│       └── news-2.html
│
├── catalog/                    # Раздел Каталога Станков (Директория-обертка)
│   ├── category-1/             #   - Папка Категории (например: cnc-milling - Фрезерные)
│   │   ├── product-1/          #     - Папка Товара (например: innos-h130)
│   │   │   ├── assets/         #       - Локальные фото, чертежи, PDF только для этого станка
│   │   │   └── product-1.html  #       - Карточка станка
│   │   └── product-2/          #     - Второй товар в этой же категории...
│   │
│   └── category-2/             #   - Вторая категория (например: cnc-turning - Токарные)
│       └── product-3/
│           ├── assets/
│           └── product-3.html
│
├── about.html                  # КОРНЕВАЯ СТРАНИЦА: О компании
├── catalog.html                # КОРНЕВАЯ СТРАНИЦА: Главная каталога
├── contacts.html               # КОРНЕВАЯ СТРАНИЦА: Контакты
├── index.html                  # КОРНЕВАЯ СТРАНИЦА: Главная страница сайта
└── news.html                   # КОРНЕВАЯ СТРАНИЦА: Журнал/Список новостей
```

## 2. Жесткие правила работы с медиа-файлами (Изображения)

Для абсолютной независимости от внешних облачных сервисов и удобства редактирования ИИ и людьми:

* **СТРОГИЙ ЗАПРЕТ ВНЕШНИХ ССЫЛОК:** Ни в одном HTML-файле проекта (ни в корневых страницах `index.html`, `about.html`, ни во вложенных карточках `product.html`) не должно быть внешних ссылок на изображения (например, `<img src="https://...">`). Любая используемая картинка или фон должны быть немедленно скачаны и сохранены локально.
* **Формат файлов:** Абсолютно все графические файлы (фотографии оборудования, фоновые текстуры, элементы интерфейса, схемы), размещаемые в проекте, ДОЛЖНЫ быть сохранены исключительно в формате **.PNG**. Использование WebP/JPG запрещено во избежание потери качества при пересохранении дизайнера.
* **Правила размещения (Global vs Local):** 
  - Картинки, используемые на общих корневых страницах (`index`, `catalog`, `about`, `contacts`, `news`), должны храниться в глобальной директории: `/innos-site/assets/`.
  - Изображения, относящиеся к конкретной вложенной сущности (станок или новость), ОБЯЗАНЫ лежать только в своей локальной директории (например: `/catalog/category-name/product-name/assets/`). Их категорически запрещено сваливать в общую папку. Это гарантирует модульность: станок можно скопировать или удалить вместе с его папкой без забытых медиа-ресурсов.

## 3. Frontend Dependency Policy (ОБЯЗАТЕЛЬНО)

Проект должен быть устойчив к блокировкам внешних ресурсов. Поэтому принята политика **100% локальных фронтенд-зависимостей**.

### 3.1 Что запрещено
- Подключение внешних CSS/JS из CDN:
  - `<script src="https://...">` для UI-библиотек;
  - `<link href="https://...">` для CSS-фреймворков.
- Подключение внешних шрифтов и иконок (Google Fonts, external icon CDN).
- Использование runtime-framework CDN в разметке (например, Tailwind CDN script в HTML).
- Любые внешние ассеты, без которых страница ломает стиль/логику.

### 3.2 Что разрешено
- Только локальные файлы:
  - `/innos-site/assets/css/*.css`
  - `/innos-site/assets/js/*.js`
  - `/innos-site/assets/fonts/*` (если используются web-fonts)
  - `/innos-site/assets/icons/*` или локальные SVG-спрайты.
- Чистый стек: `HTML + CSS + JS` без обязательного сборщика.

### 3.3 Базовый стандарт подключения
- На каждой странице стили и скрипты подключаются только локально:
  - `<link rel="stylesheet" href=".../assets/css/main.css">`
  - `<script defer src=".../assets/js/main.js"></script>`
- Перед коммитом обязательно проверить отсутствие внешних зависимостей:
  - `rg -n "https://|http://" innos-site/*.html innos-site/**/*.html`

## 4. SEO-соглашения
- Названия папок (`product-1`, `category-1`, `news-1`) должны транслитерироваться (например, `frezerny-stanok-v500`). 
- Использование пробелов в названиях папок и файлов строго запрещено — только дефисы (`-`). 
- Сохраняйте вложенность для хлебных крошек (Breadcrumbs). Страница станка должна отражать путь `Главная -> Каталог -> Категория -> Станок`.
