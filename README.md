# Проект: Станки Иннос (Innos Website)

**Ключевая цель проекта** — создание высококонверсионного B2B-портала по продаже и сервису тяжелого промышленного оборудования (станков с ЧПУ, 5-осевых портальных центров). Проект уходит от "классического" маркетингового лендинга в сторону "Экспертного Индустриального Дашборда".

---

## 🛑 ОБЯЗАТЕЛЬНЫЕ ПРАВИЛА (AI CORE DIRECTIVES)
При любой генерации кода, создании новых страниц, изменении архитектуры файлов или работе с текстом — нейросеть (AI) обязана **СТРОГО следовать документам**, перечисленным ниже. Любое отступление от этих правил является критической ошибкой.

### 1. Архитектура и файловая структура
👉 **Файл:** `ARCHITECTURE_RULES.md`
- Запрет на внешние ссылки для графики (`<img src="http...">` запрещено). Все картинки хранятся только локально!
- Строгий формат медиа-файлов — только `.PNG`.
- Логика Flat-Tree/Silo для вложенных папок: изображения корневых страниц лежат в `/assets/`, а карточек товаров — изолированно в их собственных папках `.../product-name/assets/`.
- Проект работает в режиме **offline-first**: любые внешние CDN и внешние зависимости для фронтенда запрещены.

### 2. Дизайн-система и Визуальный код (The Machining Monolith)
👉 **Файл:** `DESIGN.md`
- Темный, "дорогой" интерфейс (`#131313`). Цвета: Электрик Блю, Металлический Серый.
- Правила Glassmorphism (темное полупрозрачное стекло с размытием).
- Сетка, отступы, шрифты (Space Grotesk для заголовков, Inter для таблиц и ТТХ).
- Отказ от сплошных разделительных линий в пользу изменения тональности секций.
- Анимации загрузки "из темноты" и Tonal Depth.
- Техническая реализация дизайна только на локальных `HTML/CSS/JS` без React/Vue/Angular и без runtime CSS-CDN.

### 3. SEO-Протокол и Логика Аудита B2B
👉 **Файл:** `UX_SEO_AUDIT_PROTOCOL.md`
- Инструкция о том, КАК анализировать текст и структуру (роль Эксперта по B2B CRO).
- Приоритет фактов (шпиндель, ±0.002 точность, SLA 24/7) над абстрактной маркетинговой водой ("надежные технологии будущего").
- Обязательное присутствие "Коммерческих факторов ранжирования Яндекса" (INN/OGRN, телефоны, адреса складов).

## Памятка разработчикам и AI:
1. При начале любой новой сессии — предварительно выполняйте чтение этих трех `*.md` документов через инструмент `view_file`.
2. Когда создаете новые страницы (например, 20 новых карточек для 20 станков), каждая должна строго наследовать дизайн-систему из `DESIGN.md` и файловую структуру из `ARCHITECTURE_RULES.md`.
3. Для текстов — используйте инженерный tone-of-voice и рекомендации из `UX_SEO_AUDIT_PROTOCOL.md`.
4. Любая новая внешняя ссылка на фронтенд-зависимости (`cdn`, `fonts`, `js/css` с внешнего домена) считается критической ошибкой.
5. Перед коммитом обязательно запускать локальную проверку:
   - `./scripts/check_local_frontend.sh innos-site`
